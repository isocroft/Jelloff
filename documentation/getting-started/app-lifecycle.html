  • §Application Startup

    At startup, Jollof makes use of a .htaccess file to call up the entry point file public/index.php which loads up all class loaders for the application (Composer and Jollof autoload files). This files are necessary to make all Jollof Core & Extension APIs available.

    The core application class is instantiated at this point too to provide access to other functionalities.

  • §Services Installation

    Thereafter, the core application object instantiated in the prior step is used to register 2 major config services for the application. The db and the env services. This loads up necessary configurations for Database connections and Components Setups utilized in the next step.

  • §Componets Registration

    At this point, the core application components are registered (loaded into a container sort of) and are primed with the necessary configurations to make them work. Components like the Router, Session, Logger, Comms e.t.c are registered here.

  • §Middlewares Setup

    Next, The middlewares for the application are loaded up so the application can iterate over them one after another to determine if the current requested will be completely serviced or truncated abruptly.

  • §Events Setup

    Next, The events for the application are loaded up too to provide a central mediator construct for handling both application-level events (e.g. ERRORS, UNKNOWN ROUTES) and custom events which are created to achieve some purpose in the application. Custom events are created with the ::on( ... ); method of the System component. See System Component reference for more details.

  • §Routes Setup

    Subsquently, all application routes are loaded up and stored in a routes table from where the current route is determined. Each defined route have several properties that go along with it. These properties serve as some form of filters for them and when any filter fails, request handling is terminated in an fatal error.

    The properties include the verb, the models, the params and the ajax property. The verb property serves to dictate the HTTP method that will be used for accessing the route (e.g PUT, GET, HEAD, POST, DELETE). The models property is an array of model classes to to be instatiated for use in the controller. The params property is an array of route parameter names and the regular expressions used to validate them. Finally, the ajax property is a flag to dictate when the route is to be accessed via AJAX or not.

    Routes are created using the ::bind( ... ); method of the Router component. See Router Component reference for more details.

  • §Controller Instantiation

    Immediately after the current route is detected, the corresponsing controller is located and instantiated (as a class) and then the appropriate method called. The first part of a route is directly related to the controller to be instantiated and the second part is related to the controller method to be called. the third and subsequent parts of the route are related to the route parameters.
