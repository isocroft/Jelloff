  • § Setup View File

    Setting up a view entails creating a view file by:

    1. using the console command for view creation.
    2. manually creating a view file using the OS GUI or terminal.

    Views are triggered whenever the Response::view(); api is used within a controller method. A compilation process (i.e. the conversion of .view files to .php files) is always completed before view contents are sent to the client-side as part of the response.

  • § View Template Syntax

    Jollof views have a syntax which makes it possible to build out completly using the view data passed into the view file.

    1. For string, integer and boolean type view data, precede the view data name by =@.
    2. For array type view data, the [loop:] construct can be used to access its' contents at each index.
    3. The [if:] construct can be used to setup a conditional statement for array, string, integer and boolean type view data.
    4. The [choose:] construct can be using to switch on cases that apply to the array, string, integer and boolean type view data given.

    [loop: <array> ] Example Usage: 
       [loop:@posts]
          <a href="[!url(/postings/=@branch)]" id="[@posts_index]"> [@posts_value] </a>
   [/loop]
    [if: <array|string|integer> ] Example Usage: 
       [if:@alltasks <= 23]
          <div class="ramp"> Too low... </div>
   [elseif:@alltasks > 90]
          <div class="ramp"> Off the charts! </div>
   [/if]
