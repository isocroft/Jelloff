  • ยง Request - system/base/components/Request

    This component is responsible for handling all HTTP requests.

    ::uri( void ); This will retrieve the pathname of the current HTTP request Example Usage: 
       
    
        $uri = Request::uri();
    ::method( void ); This will retrieve the current HTTP request method Example Usage: 
       
     
        $method = Request::method();
    ::getAuth( void ); This will retrieve the current HTTP request Basic OR Digest authorization credentials Example Usage: 
       
     
        $authString = Request::getAuth();
    ::getOrigin( [bool] ); This will retrieve the current HTTP request origin Example Usage: 
       
     
        $endpoint = Request::getOrigin();
    ::getPort( void ); This will retrieve the current server port number Example Usage: 
       
     
        $port = Request::getPort();
    ::getProtocol( void ); This will retrieve the current server scheme/protocol Example Usage: 
       
     
        $scheme = Request::getProtocol();
    ::getCookie( string ); This will retrieve the current HTTP request cookie value for a given cookie name as a string Example Usage: 
     
       
        $sessionId = Request::getCookie('_my_cookie');
    ::ip( void ); This will retrieve the remote client ip address Example Usage: 
       
     
        $ip_address = Request::ip();
    ::contentType( void ); This will retrieve the content type for the current HTTP request entity body. Example Usage: 
       
     
        $type = Request::contentType();
    ::referer( void ); This will retrieve the referer url/pathname for the current HTTP request Example Usage: 
      
     
         $referer = Request::referer();
    ::upload( string [, &array ] ); This will upload client files to the destination server as setup in the config section for application uploads Example Usage: 
         
        $uploadErrors = array();
        $statusArray = Request::upload('display_photos', $uploadErrors);
    ::isAjax( void ); This will check if/whether an HTTP request is an asynchronous (AJAX) one Example Usage: 
      
                 
                     $isAsync = Request::isAjax();
    ::isPjax( void ); This will check if/whether an HTTP request is a pushState asynchronous (AJAX) one Example Usage: 
      
                 
                     $pJax = Request::isPjax();
    ::input( void ); This will retrieve all GET, POST, PUT, DELETE, PATCH inputs to the request. The input methods serves to collect both files and fields status: DEPRECATED Example Usage: 
     
     
      /* get the request body (object) */
      $body = Request::input();

      /* 
            get all files attached to the request 
            - POST/PUT requests 
       */
      $files = $body->getFiles(); 

      /* 
            get all fields attached to the request
            - GET/POST/PUT/PATCH 
       */
      $fields = $body->getFields();
    ::getDeviceType( void ); This will reveal the type of the device which the request originated from Example Usage: 
     
       
        $device = Request::getDeviceType();

  • ยง Response - system/base/components/Response

    This component is responsible for handling all HTTP responses.

    ::json( array, integer ); This will send JSON data back to the client.The second integer type argument is optional Example Usage: 
     
     
        /* Return an unauthorized status (401) with JSON data */
       Response::json(array('status' => 'not-ok'), 401);
    ::status( integer ); This will send the set HTTP status back to the client Example Usage: 
     
     
       Response::status(401);
    ::redirectBack( void ); This will instruct the client to redirect to the previous route (the previous route is tracked using the session) Example Usage: 
     
       
        Response::redirectBack();
    ::text( string ); This will send PLAIN TEXT data back to the client Example Usage: 
     
       
        $text = 'Hello World!';
        Response::text($text);
    ::view( string [, array [, array]] ); This will send markup (HTML or XML) data back to the client. The second array type argument is optional. When passed it feeds data to Jollofs' Templating renderer. Example Usage: 
     
      
        /* path to a HTML view */
        $viewHTMLPath = 'app/dashboard'; 
        Response::view($viewHTMLPath, array('id' => 12345678));
        
        /* path to an XML view - Twillio TwiML */
        $viewXMLPath = 'twillio/callxml';
        Response::view(
                $viewXMLPath, 
                array('calltext' => 'Hello, How are you?'), 
                array('asXML' => TRUE)
        );

        /* Setting up HTTP/2 Server-Push for CSS files */
        Response::view(
                $viewHTMLPath,
                array('posts' => array()),
                array('serverPush' => '.css')
        );
    ::file( string ); This will send contents of the given file back to the client Example Usage: 
     
        
        $imgefile = 'bitmap.png';
        Response::file($imagefile);
    ::download( string ); This will display the prompt window on the client for a file download. This method interacts with files in the "download" and "upload" folders found in the "storage/cabinet" path Example Usage: 
     
       
        $file = 'careers.pdf'; 
        Response::download($file);
    ::isEmpty( void ); This will report whether or not the response has an entity body. Example Usage: 
     
       
        /* 
            GET responses are usually not empty 
            - i.e. GET responses always have an entity body 
        */
        $isGETResponse = !Response::isEmpty();
    ::setCookie( string, mixed ); This will set a cookie on the response to the client. when the data type passed as the cookie value is not a "string", it is serialized as JSON. Example Usage: 
     
       
        $details = array('username'=> 'eniobong', 'id' => '23jwn47uya663q3e'); 
        Response::setCookie('USERPROFILE', $details);
    ::header( string, string ); This will setup a given HTTP response header name with a value for the client response Example Usage: 
     
       
        Response::header('Cache-Control', 'private');
    ::redirect( string ); This will redirect the client to the route path passed in as first argument Example Usage: 
     
       
        $route = '/campaigns/tv'; 
        Response::redirect($route);

  • ยง Session - system/base/components/Session

    This component is responsible for handling sessions and session data.

    ::get( string ); This will retrieve the value for a given session data name. Example Usage: 
     
       
        $data = Session::get('user_data');
    ::forget( string ); This will delete the value and associated session data name for a given session data name. Example Usage: 
     
      
       $forgotten = Session::forget('user_data');
    ::token( void ); This will retrieve the csrf token for the given HTTP request Example Usage: 
     
       
        $token = Session::token();
    ::has( string ); This will determine whether the current session has the a value for the given key Example Usage: 
     
      
        $has = Session::has('accessControl');
    ::drop( void ); This will destroy the entire session and its associated data Example Usage: 
     
       
      Session::drop();

  • ยง System - system/base/components/System

    This component is responsible for handling application-wide eventing and requests filtering (via Middlewares).

    ::on( string , callable ); This will setup/register an observer and/or event handler for a given custom event triggered using the ::fire( ... ); method. Example Usage: 
          
      System::on('userRegistered', function($eventData){
			# code ....
      }); 

      System::on('rewriteFailed', create_function('$data', ' /* code .... */ '));
    ::fire( string [, mixed [ ,..]] ); This will trigger the observer and/or event handler for the specified custom event registered using the ::on( ... ); method. The second mixed type argument is optional. The return value of the callable from the ::on( ... ); call will be passed back from the ::fire( ... ); method call. Example Usage: 
     
       
        $value = System::fire('userRegistered', array('timestamp'=>time()));
    ::middleware( string , callable ); This will register an application-wide conditional filter for all HTTP requests. The second callable argument must return a boolean value to notify the application of the status of the filter operation. Example Usage: 
     
       
        System::middleware('verifyCORSAccess', function($currentRoute, $auth){
            # $currentRoute - This is the currentRoute for the request being served
            # $auth - This is the authorization component object(Auth) that manages authroization for routes based on active sessions
        
            # code ...
        });

  • ยง Logger - system/base/components/Logger

    This component is responsible for handling application logging on any level (ERROR, WARNING & INFO) for any event or execution point.

    ::info( string [, string ] ); This will write the given text to a log file (e.g. exec.log) on an INFO level. The second string type argument is optional. Example Usage: 
        
      Logger::info("variable value: " . isset($accountNumber));
    ::error( string [, string ] ); This will write the given text to a log file (e.g. exec.log) on an ERROR level. The second string type argument is optional. Example Usage: 
         
      Logger::error("Wahala dey! The redis db connection couldn't be made");
    ::warn( string [, string ] ); This will write the given text to a log file (e.g. exec.log) on an WARNING level. The second string type argument is optional. In the example below, the second string argument is given a value. This value is the name of a user-created log file called warnings Example Usage: 
       
     
      Logger::warn("The database query returned NULL", "warnings");

  • ยง File - system/base/components/File

    This component is responsible for handling application filesystem operations (CREATING, READING & WRITING to files on disk or over a network).

    ::readStream( string [, array ] ); This will read the contents of the given file as a stream. The second array type argument is optional. In the example below, the second array argument is given a value. This value is an array that sets up some HTTP options. Whenever you read files off the hard disk, it is better to use the ::read( ... ); method instead. Example Usage: 
     
        
        // Create a stream
        $options = array(
          'http'=>array(
            'method'=>"GET",
            'header'=>"Accept-language: en\r\n" . "Cookie: foo=bar\r\n"
          )
        );

        // Start the web scrapper by reading the JUMIA home page. 
        $html = File::readStream("http://www.jumia.com.ng/", $options);
    ::readChunk( string [, integer] ); This will read out a file in chunks when given an offset as the second argument.The second argument is optional Example Usage: 
     
       
        $chunk = File::readChunk("readme.txt", 8);
    ::writeChunk( string , string [, integer [, bool]] ); This will write the contents of the given file. The second string type argument serves for the chunked content. The third and fourth arguments are optional Example Usage: 
     
       
        $bytesWritten = File::writeChunk(realpath("readme.txt"), "New Text Content");
    ::makeFile( string ); This will create a file with the given file name and file extension. Example Usage: 
     
       
        File::makeFile("readme.txt");
    ::grepFiles( string ); This will find all file names that match the pattern given as the first argument. Example Usage: 
     
       
        $allFilesNames = File::grepFiles("*.txt");
    ::deleteFile( string ); This will delete a file with the given file name and file extension. Example Usage: 
     
         
       File::deleteFile("readme.txt");
    ::makeFolder( string [, bool ] ); This will create a folder with the given file name and file extension. The second bool type argument is optional. When supplied as TRUE, it will hide the folder. Example Usage: 
     
       
       File::makeFolder("shelf");
    ::read( string [, bool]); This will create a folder with the given file name and file extension. The second bool type argument is optional. When supplied as TRUE, it will hide the folder. Example Usage: 
      
     
       $content = File::read("readme.txt");
    ::write( string , string [, bool] ); This will write to a file with the given file name and file extension line by line. The third bool type argument is optional. When supplied as TRUE, it will overwrite the prior contents of the file. Example Usage: 
     
       
       File::write("readme.txt", "The First Line", true);

  • ยง Router - system/base/components/Router

    This component is responsible for handling application routes.

    ::bind( string [, array ]); This will bind and/or register a given route for the application. It is only used in the routes/setup.php file. The second array type argument is optional. When supplied, it supplies all the necessary route properties. Example Usage: 
      
      
     Router::bind("/settings/basic", array('verb' => 'get', 'models' => array(), 'ajax' => true));
    ::resolve( string [, array ] ); This initiates a controller action (by calling up the appropriate controller method) as specified along with the request parameters. The second array type argument is optional. When supplied, it specifies the request parameters. Example Usage: 
     
       
      Router::resolve("/app/filter/medium", array('post' => array('key' => '2873652672')));

  • ยง Cache - system/base/components/Cache

    This component is responsible for handling application cache (mostly powered by memcached).

    ::has( string , string ); This checks if a given cacheable resource is available for retrieval or modification. Example Usage: 
     
      
     $wasCached = Cache::has("lineburst");
    ::get( string ); This retrieves a given cacheable resource via its key. Example Usage: 
     
      
      $value = Cache::get("lineburst");
    ::set( string , string ); This sets a cacheable entry given a key. Example Usage: 
     
      
      Cache::set("lineburst", (new \stdClass()));

  • ยง Comms - system/base/components/Comm

    This component is responsible for handling application communication of all sorts (e.g sending text messages, making calls, sending emails). At the background, it uses Twilio for sending SMS and making Calls and Mailgun or PHPMailer for sending emails. It also uses Pusher for sending realtime messages.

    ::sendBroadcast( string , string, mixed ); This sends a realtime message using the underlying Pusher composer library installation. Example Usage: 
     
      
      // Sending a realtime  message (powered by Pusher).

      $data = array('id' => 1, 'msg' => 'I\'ll look at it now');
      Comms::sendBroadcast("channel-1", "forumMsg", $data);
    ::sendMessage( string , string ); This sends a text message or SMS to the number provided as the first argument. The second argument is for the message body itself. Example Usage: 
     
      
      // Sending a text message (powered by Twilio).

      Comms::sendMessage("09034560000", "See bros, pay up your invoice abeg.");
    ::sendCall( string , string ); This sends a call to the number provided as the first argument. The second argument is for the URL to the TwiML xml file which should contain the words for the call. Example Usage: 
     
      
      // Making a call (powered by Twilio).

      Comms::sendCall("09034560000", "");
