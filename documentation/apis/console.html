  • § make:env - arguments: none

    This console command will enable the creation of an .env file for the application. This file contains the application secret and all database credentials.

    Example Usage: 
     >  php jollof make:env

  • § make:component - arguments: <Component_Name>

    This console command will enable the creation of a custom Component for the application.

    Example Usage: 
     >  php jollof make:component ErrorReporter

  • § make:view - arguments: <View_Name>

    This console command will enable the creation of a View file for the application. In the example below, a home file is created inside a master folder. If the

    Example Usage: 
     >  php jollof make:view master/home

  • § make:model - arguments: <Model_Name>

    This console command will enable the creation of a Model file for the application.

    Example Usage: 
     >  php jollof make:model Posts

  • § make:controller - arguments: <Controller_Name>

    This console command will enable the creation of a Controller file for the application.

    Example Usage: 
     >  php jollof make:controller Accounts

  • § make:route - arguments: <Route_Url>, <Route_Verb>

    This console command will enable the creation of a Route for the application.

    Example Usage: 
     >  php jollof make:route /login PUT

  • § run:tasks - arguments: <Task_Name>

    This console command will enable the processing for npm grunt related tasks for the application.

    Example Usage: 
     >  php jollof run:tasks concatmin

  • § run:tests - arguments: none

    This console command will run all unit tests using the composer phpunit package for the application.

    Example Usage: 
     >  php jollof run:tests
